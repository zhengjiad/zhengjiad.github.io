collection的结构

collection 包括List列表和Set集

Collection是单列集合

List

  1. List 元素是有序的可重复的
  2. 有序的List 对元素的插入位置可以精确的控制。
  3. 可以根据元素的索引访问元素。
  4. list适合经常插入，删除数据，但是随机取数的效率很低。

Set

  1. 无序的不可以重复的。
  2. 取出元素的方法只有迭代器。
  3. 不可存放重复元素，元素存取是无序的。
  4. 适合经常的随机存储，插入，删除。但是在遍历的时候效果不好。

list和set下层

LIST

  • vector 线程安全，但是速度慢，已被ArrayList代替。底层数据结构是数组结构。
  • ArrayList 线程不安全，查询速度快。底层数据结构是数组结构。
  • LinkList 线程不安全，增删速度快。底层数据结构是列表结构。

Set

  • HashSet 线程不安全，存取速度快。依赖元素的hashcode方法和equal方法保证元素的唯一性。
  • TreeSet 线程不安全，可以对set集合中的元素进行排序。通过compareTo或者compare方法保证元素的唯一性。元素是以二叉树的形式存放的。

collection和collections的区别

  • collection是集合类的上级接口，子接口主要有Set，List，Map
  • collections是针对集合类的一个帮助类，提供了操作集合的工具方法，一系列静态方法实现对集合的搜索，排序，线程安全化等操作。
