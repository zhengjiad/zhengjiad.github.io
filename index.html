选择类排序

思想：每一次遍历在n-i+1(i=1,2,……,n-1)个记录中选取关键字最小的记录作为有序序列的第i个记录。

简单选择排序

  1. 思想：第一次遍历时，从第一个记录开始，通过n-1次关键字的比较，从n个记录中选取关键字最小的记录，并和第一个记录进行交换。第二趟从第二个记录开始，选择最小的记录和第二个记录交换。以此类推，直到全部排序完毕。
  2. 时间复杂度：O(n^2)
  3. 空间复杂度 ：O(1)
  4. 稳定性：不稳定排序
  5. 程序：

树形选择排序

  1. 思想：为了减少比较次数，两两进行比较，得出的较小的值再两两进行比较，直至得出最小的输出，然后在原来位置上置为∞，再进行比较。直至所有都输出。
  2. 时间复杂度：O(nlogn)
  3. 空间复杂度：O(n)
  4. 稳定性：稳定排序
  5. 程序：

堆排序

  1. 思想：把待排序的关键字存放在数组r[1…n]中，将r看成是一棵完全二叉树的顺序表示，每一个节点表示一个记录，第一个记录r[1]作为二叉树的根，下一个记录r[2…n]依次逐层从左到右顺序排列，任意节点r[i]的左孩子是r[2i]，右孩子是r[2i+1]，双亲是r[i/2向下取整]。然后对这棵完全二叉树进行调整建堆。
  2. 时间复杂度：O(nlogn)
  3. 空间复杂度 ：O(1)
  4. 稳定性：不稳定性排序。{5,5,3}
  5. 程序：

归并排序

归并排序

  1. 思想：假设初始序列有n个记录，首先将这n个记录看做n个有序的子序列，每个子序列长度为1，然后两两归并，得到n/2向上取整个长度为2(n为奇数时，最后一个长度为1)个有序子序列。在此基础上，再对长度为2的有序子序列进行两两归并，得到若干个长度为4的有序子序列。如此重复，直到得到一个长度为n的有序序列为止。
  2. 时间复杂度：O(nlogn)
  3. 空间复杂度：O(n)
  4. 稳定性：稳定排序
  5. 程序：

分配类排序

分配类排序

思想：分配类排序是利用分配和收集两种基本操作。
分类：

  • 多关键字排序
  • 链式基数排序
  1. 思想：先分配再收集，就是按照一个次关键字分配一下，然后进行收集，然后再换个关键字把序列再分配一下，再进行收集，完成一次排序，这样所有关键字分配收集完毕后，就完成了排序。
  2. 时间复杂度：O(d(n+rd))
  3. 空间复杂度：O(rd)
  4. 稳定性：稳定排序
    5.程序

总结

  • 简单排序法一般只用于n较小的情况（例如n<30）。当序列的记录“基本有序”时，直接插入排序是最佳的排序方法。如果记录中的数据较多，则应采用移动次数较少的简单选择排序法。
  • 快速排序、堆排序和归并排序的平均时间复杂度均为O(n㏒n)，但实验结果表明，就平均时间性能而言，快速排序是所有排序方法中最好的。遗憾的是，快速排序在最坏情况下的时间性能为O(n²)。堆排序和归并排序的最坏时间复杂度仍为O(n㏒n)，当n较大时，归并排序的时间性能优于堆排序，但它所需的辅助空间最多。
  • 可以将简单排序法与性能较好的排序方法结合使用。例如，在快速排序中，当划分子区间的长度小于某值时，可以转而调用直接插入排序法；或者先将待排序序列划分成若干子序列，分别进行直接插入排序，然后再利用归并排序法，将有序子序列合并成一个完整的有序序列。
  • 基数排序的时间复杂度可以写成O(d·n)。因此，它最适合于n值很大而关键字的位数d较小的序列。当d远小于n时，其时间复杂度接近O(n)。
  • 从排序的稳定性上来看，在所有简单排序法中，简单选择排序是不稳定的，其他各种简单排序法都是稳定的。然而，在那些时间性能较好的排序方法中，希尔排序、快速排序、堆排序都是不稳定的，只有归并排序、基数排序是稳定的。

基本概念

  1. 排序：按照一定的关键字，将一个序列排列成一个想要得到的新序列。
  2. 内部排序和外部排序：整个排序过程完全在内存在进行，叫做内部排序。数据量较大需要借助外部存储设备才能完成，叫做外部 排序。
  3. 主关键字和次关键字：
  4. 排序的稳定性：对于相同的元素来说，在排序之前和之后的记录是一样的，那么就是一种稳定的排序，如果顺序发生变化就是不稳定的排序。

插入类排序

思想：在一个已经排序好的序列中，将未被排进的元素按照原来的的规定插入到制定位置。
分类：

直接插入排序：

  1. 思想：最基本的插入排序，将第i个插入到前i-1个中的适当位置。
  2. 时间复杂度：T(n)=O(n^2)
  3. 空间复杂度：S(n)=O(1)
  4. 稳定性：稳定排序。 循环条件while(r[0].key< r[j].key)保证。
  5. 程序：

折半插入排序

  1. 思想：因为已经确定了前半部分是有序序列，所以在查找插入位置的时候可以用折半查找的方法进行查找，提高效率。
  2. 时间复杂：比较时的时间减为O(nlogn)，但是移动元素的时间耗费未变，所以总的时间复杂度还是O(n^2)。
  3. 空间复杂度：S(n)=O(1)。
  4. 稳定性：稳定排序。
  5. 程序：

希尔排序

  1. 思想：又称缩小增量排序法。把待排序序列分成若干较小的子序列，然后逐个使用直接插入排序法排序，最后再对一个较为有序的序列进行一次排序，主要是为了减少移动的次数，提高效率。原理应该就是从无序到渐渐有序，要比直接从无序到有序移动的次数少一点。
  2. 时间复杂度：O(n^1.5)
  3. 空间复杂度：O(1)
  4. 稳定性：不稳定排序。{2,4,1,2}，2和1一组，4和2一组，进行希尔排序，第一个2和最后一个2会发生位置上的变化。
  5. 程序：

交换类排序

思想：通过交换逆序元素进行排序的方法
分类：
冒泡排序：

  1. 思想：反复扫描待排序序列，在扫描过程中顺序比较相邻的两个元素的大小，若逆序就交换顺序，第一遍，从第一个数据开始，比较相邻的两个数据，（以升序为例）如果大就交换，得到最大的数据在末尾，然后进行第二趟，只扫描前n-1个元素，得到次大的放在倒数第二位。以此类推，最后得到升序序列。如果在扫描过程中，发现没有交换，说明已经排好序列，直接终止扫描。所以最多进行n-1次遍历。
  2. 时间复杂度：O(n^2)
  3. 空间复杂度：O(1)
  4. 稳定性：稳定排序
  5. 程序：

快速排序

  1. 思想：冒泡排序一次只能消除一个逆序 ，为了能一次消除多个逆序，采用快速排序。以一个关键字为轴，从左至右依次进行对比，然后交换，第一趟结束后，可以把序列分为两个子序列，然后再分段进行快速排序，达到高效。
  2. 时间复杂度：平均O(nlogn)，最坏O(n^2)
  3. 空间复杂度：O(nlogn)
  4. 稳定性：不稳定排序。{3,2,2}
  5. 程序：

tomcat目录结构

Veröffentlicht am 2016-08-22

tomcat7目录结构功能说明如下

  • bin:存放tomcat的一些可执行文件，比如启动tomcat或者关闭tomcat。
  • common:存放了tomcat和所有web应用都可以使用的类和jar包。
  • conf:存放tomcat的一些配置文件。
  • logs:存放tomcat和web应用输出的日志文件。
  • server:存放只有tomcat才能使用的类和jar包,以及tomcat自带的web应用。
  • temp:存放一些临时文件。
  • webapps:存放开发的web应用。
  • work:存放jsp相关的一些servlet类.

AOP

  • AOP面向切面编程（Aspect-Oriented Programming）:是一种新的方法论，是对传统OPP面向对象编程的补充。
  • AOP的主要编程对象是切面。
  • AOP的好处：
    1. 每个事物逻辑位于一个位置，代码不分散，便于维护和升级
    2. 业务模块更加简洁，只包含核心业务代码。

AOP叫做面向切面编程，顾名思义它是在核心代码的外层包裹上一层新的代码，通过这层代码对核心代码进行扩展。
